Release history

VibePod releases so far: from first agents to Pi support.

VibePod has grown from a simple local launcher into a small ecosystem for running coding agents in isolated containers, recording activity locally, adding reusable skills, and sharing updates across multiple repositories.

May 31, 2026 · CLI · Release notes

This post summarizes the CLI release line so far, based on the repository history and release tags. It is meant as a human-readable map of what changed, not a replacement for the commit log.

The arc so far

The early releases focused on making local agent containers easy to start. Later releases added logging, project configuration, image automation, IKWID mode, LLM routing, long-lived setup workflows, attach/stop improvements, and reusable skills. The current 0.13.0 work adds Pi as another supported coding agent and updates the public website and documentation around it.

0.1.0 — Initial CLI and first extra agents

The first tagged release established the VibePod CLI project structure and added Copilot and Codex agent support. This set the core direction: run coding agents locally through one command-line interface.

0.2.x — Session capture, proxy logging, and container fixes

The 0.2 releases added request/response and session capture, then tightened proxy file permissions and container attribution. Follow-up fixes improved credential handling for Copilot and Auggie and added fixes for Mistral Vibe / Devstral containers.

0.3.x — Networks, docs, PyPI install flow, and proxy ports

VibePod gained custom Docker network support, project network detection, CI/static-check documentation, PyPI installation docs, and follow-up proxy/docs publishing fixes.

0.4.0 — Project config and better agent discovery

This release added vp config init, agent-specific config bootstrap, container init scripts, short names in vp list, custom image documentation, multiple-workspace listing, and CLI previews in the docs.

0.5.x — Image automation and dashboard reliability

The 0.5 line added automatic image fetching, changed default Claude and Codex images, added CI validation for built-in images, documented detached containers, and improved Datasette startup by pulling the latest dashboard image and waiting for readiness before opening the browser.

0.6.0 — Latest-image behavior, X11 image pasting, and README refresh

VibePod refined auto-pull behavior for agent, proxy, and Datasette images, added an image-paste workflow through the X11 clipboard, refreshed the README, and added the MIT license.

0.7.0 — IKWID mode

IKWID mode introduced an agent auto-approval workflow. Later releases extended IKWID support across more agents and documented support differences.

0.8.x — LLM server integration

The 0.8 releases added support for connecting agents such as Claude and Codex to local or remote LLM servers, including Ollama and vLLM. Follow-up releases fixed auth token forwarding and required Codex flags for IKWID.

0.9.x — Agent image namespace, Gemini fixes, icons, and broader IKWID docs

VibePod moved remaining agent containers under the VibePod namespace, fixed Gemini CLI startup issues, added the project icon and tool thumbnails, added funding links, and extended IKWID support/docs for more agents.

0.10.0 — Ollama/Linux docs and packaging polish

The 0.10 release improved Ollama connection behavior on Linux, extended integration docs, implemented explicit directory permissions, and polished PyPI image/link rendering.

0.11.0 — Startup arguments and Claude setup token workflow

This release documented and tested forwarding agent-specific startup arguments and added a Claude setup-token workflow for generating long-lived tokens.

0.12.0 — Attach/stop workflows and reusable skills

VibePod added vp attach, extended vp stop to accept container names, introduced skills managers and the skills CLI, expanded skill mappings for Codex, OpenCode, and Auggie, added notes for Gemini/Copilot/Vibe, and fixed an I/O error on exit by closing URL responses.

0.13.0 — Pi agent support and website/docs refresh

The latest work adds Pi as a supported agent, fixes Pi-specific config and storage paths, updates image mappings and environment variable docs, adds a Pi screenshot, and refreshes the website and generated docs.

Why the release story spans multiple repositories

The CLI is the main user entry point, but VibePod work is intentionally split across focused repositories. Agent container definitions live in vibepod-agents, dashboard work lives in vibepod-datasette, proxy extraction lives in vibepod-proxy, and packaging/site work has its own homes too.

That means future news posts can cover a CLI release, a new agent image, a dashboard parsing improvement, a proxy data extraction change, a Homebrew update, or a reusable skills bundle. The Repositories page is the directory for those pieces.

Example commands

Across the release line, these commands show the through-line: install VibePod, run an agent, inspect logs, add skills, and switch to Pi.

pip install vibepod
vp run claude
vp logs start
vp skills list
vp run pi
